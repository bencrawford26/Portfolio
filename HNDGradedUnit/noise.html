Perlin Noise

Perlin Noise is a function used in computer graphics to produce a "smooth" randomness. Smooth in this sense meaning less random. Perlin noise works on a timescale, numbers produced at points near one another in the timescale will be similar, further away values will be more random. I have an example showing this timescale below! Ken Perlin created this function in 1985 to be able to produce more natural looking textures in computing.

I chose to start with this example because I felt like it was the best way to help understand the "timescale" Perlin noise works on. The line is drawn along the x axis as "time" moves forward and its position on the y axis is plotted according to the Perlin noise value currently being output. If you move the slider labelled noise then the amount Perlin noise moves along its timescale is changed, producing more, or less sporadic values. Moving the x speed slider has the line drawn across the x axis at a faster rate.

For load time purposes, this app and the following three only run when the mouse is clicked on them as they're very heavy on processing. This app here is 1 dimensional noise being used to fill each pixel on screen. if you move the mouse up and down whilst clicking on the canvas the detail of noise will also be changed.

This app is very similar to the one prior, however, in this app 2 dimensional noise is used. The example with 1 dimensional noise has that line effect as perlin noise only checks the values next to it. With 2 dimensional noise each pixel in every direction is checked so each pixel must be of a similar colour to every one around it.

This example uses Perlin noise to generate a smoke effect. The sliders on the left of the canvas affect the red, green and blue values of the smoke. The slider on the right increases or decreases the amount noise is incremented each loop. Have a mess around and see what kind of other effects can be generated using the sliders.

This final example is similar to the smoke effect example. There are no sliders to manipulate the colour in this one though, as the colour in this one is being generated by noise, instead of the brightness, like in the previous example. The slider in this canvas changes the amount noise is incremented each loop. It's not the most exciting example but I think the effect is nice and easy on the eyes!
